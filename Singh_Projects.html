Scratch Project

Computer Science Principles

Fall 2018

Project on Scratch
Read the Documentation

This project involved the use of scratch, a visual programming language available for everyone on the internet. Through this project we solidified our knowledge on basic programming principles such as variables by implementing them into a project of our choice. With scratch, we were allowed to make a game or a story, of which we chose a game modeled after the popular Space Invaders. Our game consisted of 2 levels, one normal and one boss level, both of which would contain enemies randomly shooting the player, the only separation being that the boss ship would move in directions while the enemies in level 1 would stay unmoving. The boss would also have a certain amount of health that the player would need to win the game. Through this project we learned fundamental programming concepts such as variable types and operators. In addition, we also learnt organization practices for a project, such as keeping multiple versions of the product, regularly updating a daily log, and to design a product in three tiers, with more features and higher workload in each tiers, which would keep our goals clear and our progress easily trackable.


How to play:

Controls

Left and Right Arrow Keys to Move
Space to Shoot

Aim of the Game

To clear both of the levels without getting hit by the enemy ships.

Space Journey

Summer 2018

Using the Unity game engine and the Blender modeling software, this was one of my first major 3D projects during the summer because I started from scratch and made all parts of the game myself. The concept of the game was simple, with a spaceship moving through space while trying to dodge the asteroids. The execution, however, was not as simple. My prior knowledge about spawning objects was inefficient and I learned many new methods to make actions efficient, such as using object pooling instead of instantiating each of the asteroids to reduce lag in the game. I also solidified my knowledge about core programming concepts such as variables, functions, and if statements.

Jumper

Summer 2018

Jumper was one of my first 2D projects that I had made with the Unity game engine. This project served as a test for my programming skills with the C# language, so many parts of the game were simplified to have it easy for me to focus on the programming. The game art was premade and was used from the Unity Asset Store and was by Bayat Games. From this project I learned about the Unity physics system as I had used physics to make character control feel realistic. I also learned a significant amount about AI in this project. Because I had a type of enemy that would follow the player, I had to program the AI to take the shortest route to the player. Overall, in addition to testing my prior knowledge about C# in Unity, I also learned some more advanced scripting concepts that would help me in my future projects.


Text Adventure

Computer Science Principles

Spring 2019

Project on Repl.it
Read the Documentation

The objective of this project was to create a working text adventure game in python. This project was conducted in January in a roughly 2 week span. This project provided the opportunity to show display our skills and knowledge of core programming concepts. My partner and I chose to do a medieval theme game, where we divided our tiers into different achievable goals within the time period. We came across several problems, such as difficulties with path management, but through much testing, were able to find solutions, such as using small functions to keep track of each section of the path. Through this project I was able to learn about the incremental and iterative process of programming, especially in the use of an applied programming language, python. We also learnt to use flowcharts to effectively manage the complexity of the program, which allowed us to easily complete the program. Our entire experience in this project reflected the use of active communication, pair programming, and leadership in parts where the other was struggling.
